# Website Redesign Design Spec
**Date:** 2026-05-12
**Project:** Darwin Del Castillo — Personal Academic Website

---

## Direction

Portfolio-Forward (white background variant), Academic-Minimal feel. Inspired by Jeremy Labrecque's site structure. White background, navy `#1b3a6b` accents, Montserrat font retained. Dark mode support architected in CSS but delivered as a future phase.

---

## Architecture

The build system moves to a **hybrid hand-written HTML + Pandoc publications pipeline**.

### File Structure

```
src/
  head-manual.html       ← hand-written: nav, bio, interests, education
  tail-manual.html       ← hand-written: recent posts, teaching & talks, footer
  styles.css             ← all CSS (enhanced)
  references.bib         ← Paperpile export (unchanged)
  bold-author.lua        ← Lua filter (unchanged)
  bold-only.lua          ← Lua filter (unchanged)
  head-pubs.html         ← publications page header (unchanged)
  foot.html              ← footer fragment (unchanged)
  csl/                   ← citation style (unchanged)

docs/
  index.html             ← assembled by `make`: head-manual + research fragment + tail-manual
  publications.html      ← auto-generated by Pandoc (unchanged workflow)
  fragments/
    research.html        ← auto-generated: Pandoc + bold-author.lua from references.bib
  styles.css             ← copied from src/styles.css
  images/                ← unchanged
```

### Makefile targets

- **`make`** (default) — regenerates `docs/fragments/research.html` from `.bib`, then assembles `docs/index.html` from the three parts
- **`make publications`** — regenerates `docs/publications.html` only
- **`make styles`** — copies `src/styles.css` → `docs/styles.css`

### Workflow for common tasks

| Task | Action |
|------|--------|
| Update bio, interests, education | Edit `src/head-manual.html` directly, run `make` |
| Add new post or teaching entry | Edit `src/tail-manual.html` directly, run `make` |
| Add new publication (Paperpile) | Export `.bib` → `src/references.bib`, run `make` |
| CSS change | Edit `src/styles.css`, run `make styles` |

The `bold-author.lua` filter continues to auto-select 1st/2nd author papers for the homepage and append "See all publications →". No changes to filtering logic.

## Design System

### Typography

| Element | Current | New |
|---------|---------|-----|
| Hero name (`h1` in about) | `1.5rem` | `2rem`, weight 700 |
| Subtitle tagline | — | `0.9rem`, `#1b3a6b`, weight 600 |
| Section labels | `h2` with left border | Small-caps uppercase, `letter-spacing: 0.05em`, `font-size: 0.65rem`, `color: #1b3a6b` |
| Body text | `0.95rem / 1.75` | unchanged |

### Color System — CSS Custom Properties

All color values move from hardcoded hex to custom properties on `:root`. This is the dark mode foundation; no dark mode styles are shipped in this iteration.

```css
:root {
  --color-primary: #1b3a6b;
  --color-text: #1a1a2e;
  --color-bg: #ffffff;
  --color-muted: #666666;
  --color-border: #e0e8f4;
  --color-surface: #f8faff;
}
```

All existing `#1b3a6b`, `#1a1a2e`, `#fff`, `#666`, `#e0e8f4`, `#f8faff` references in `styles.css` are replaced with these variables.

### Dark Mode (future phase)

A `[data-theme="dark"]` block is added to `styles.css` but left empty (commented placeholder). A moon icon button is added to the nav in `src/head-manual.html` with a small inline `<script>` that:
- Toggles `data-theme="dark"` on `<html>`
- Persists the preference via `localStorage`

The toggle is visible but has no visual effect until dark mode colors are defined — intentional, so the infrastructure is in place.

## Homepage Sections (`src/head-manual.html` + `src/tail-manual.html`)

Section order follows Jeremy Labrecque's homepage structure:

1. **Bio** (`head-manual.html`) — photo left, right column: name + credentials (`MD, MPH`), subtitle tagline line, dual affiliation line, research identity sentence (blockquote-style with left border accent), social icons, "Book a meeting" button
2. **Interests** (`head-manual.html`) — bullet list (unchanged content and structure, just repositioned below bio)
3. **Education** (`head-manual.html`) — graduation cap list (unchanged)
4. **Recent Publications** — auto-injected `docs/fragments/research.html` (Pandoc-rendered, 1st/2nd author filter, "See all publications →" link preserved)
5. **Posts** (`tail-manual.html`) — all blog entries as title + date rows; entries include optional small category badge (`Methods`, `Commentary`, `Reading`) hardcoded in HTML. This is the full posts section — no separate `blog.html` page.
6. **Teaching & Talks** (`tail-manual.html`) — all courses and seminars. Same content as today, renamed heading. No separate `teaching.html` page.

The site remains **single-page**. All sections live on `docs/index.html`. The only separate page is the existing `docs/publications.html`.
